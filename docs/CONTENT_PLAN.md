# Content Plan

This MCP will not grow from “repo exists”. It needs repeated distribution around concrete workflows.

## Positioning Angles

- Build and maintain Google Forms with agents, not manual clicking.
- Use official APIs instead of brittle browser automation.
- Turn Google Forms into a programmable intake system.
- Let AI agents inspect, rewrite, restructure, and publish forms safely.

## Content Assets To Ship

## 1. Launch Post

Format:

- one-line problem
- one-line solution
- 20-second demo
- copy-paste install/config
- first prompt example

Suggested hook:

> I built an MCP that lets agents read, edit, and publish Google Forms using the official APIs.

## 2. Demo Clip

Show this exact flow:

1. connect the MCP
2. inspect a form
3. add one question
4. publish it

If the demo does not show a real edit, it will not convert.

## 3. “Prompt Pack” Post

Publish 5 useful prompts from [PROMPTS.md](./PROMPTS.md).

This increases activation because people copy prompts faster than they read docs.

## 4. Comparison Post

Compare:

- manual Google Forms editing
- browser automation
- official API via MCP

This builds trust and clarifies why the project exists.

## 5. Integration Example

Ship one practical example such as:

- generate a client intake form from a project brief
- review responses and summarize themes
- standardize form copy across multiple forms

## Distribution Channels

- GitHub README
- X / Twitter
- LinkedIn
- MCP registries
- Reddit communities for AI agents, dev tools, and no-code automation
- short Loom demo linked from all posts

## Cadence

- Day 1: launch post + repo
- Day 3: demo clip
- Day 5: prompt pack
- Day 7: integration example
- Week 2: comparison post

## Conversion Goal

The goal is not stars. The goal is:

- people can install it
- people can run one successful prompt
- people believe it is safe enough to use on a real form
